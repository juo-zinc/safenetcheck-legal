개인정보 처리방침

한국어

본 개인정보 처리방침은 SafeNet Check(이하 “본 앱”)이 사용자 정보를 어떻게 처리하는지 설명합니다.

본 앱은 개인의 로컬 네트워크 환경 이해를 돕기 위한 도구이며, 감시 기능을 제공하지 않습니다.
사용자의 프라이버시 보호는 본 앱의 핵심 설계 원칙입니다.

1. 기본 원칙

  • 사용자 계정을 요구하지 않습니다.
  • 광고 SDK 또는 추적 도구를 사용하지 않습니다.
  • 모든 분석은 사용자 기기 내에서 수행됩니다.
  • 사용자를 직접 식별할 수 있는 개인정보를 의도적으로 수집하지 않습니다.
  • 특정 기기를 추적하거나 지속적으로 식별하지 않습니다.

2. 수집 및 처리되는 정보

본 앱은 기능 수행을 위해 제한된 기술적 정보만 처리합니다.

2.1 로컬 네트워크 환경 정보

본 앱은 로컬 네트워크 환경의 특성을 이해하기 위해 다음과 같은 정보를 처리할 수 있습니다:

  • 로컬 IP 주소
  • 게이트웨이 정보
  • 네트워크 유형
  • ARP 캐시 정보
  • mDNS / SSDP 신호 존재 여부
  • Wi-Fi 접근 지점(AP) 정보
중요
  • 네트워크 통신의 실제 내용(메시지, 파일 등)은 수집하거나 저장하지 않습니다.
  • 외부 인터넷 트래픽을 감시하거나 분석하지 않습니다.
  • 본 앱은 특정 기기를 식별하거나 추적하기 위한 목적으로 사용되지 않습니다.

2.2 BLE (Bluetooth Low Energy) 신호

본 앱은 주변 BLE 광고 신호를 감지할 수 있습니다.

처리 정보

  • RSSI(신호 세기)
  • 세션 단위 난수 기반 임시 식별자

개인정보 보호 방식

  • 실제 MAC 주소는 저장되지 않습니다.
  • 기본 설정에서는 기기 이름 및 서비스 UUID를 수집하지 않습니다.
  • 모든 데이터는 기기 내에서만 처리됩니다.
  • BLE 신호는 특정 개인 또는 기기를 식별하기 위한 용도로 사용되지 않습니다.

2.3 선택 기능: Active Discovery

사용자가 직접 활성화한 경우에만 다음 기능이 동작할 수 있습니다:

  • SSDP (UPnP) 검색
  • mDNS 서비스 탐색
  • WS-Discovery (예: ONVIF)
  • 로컬 네트워크 내 응답 여부 확인

이 기능은 다음 조건을 따릅니다:

  • 로컬 사설 IP 범위에서만 수행됩니다
  • 외부 인터넷 서버를 대상으로 하지 않습니다
  • 연결 가능 여부 및 서비스 존재 여부만 확인합니다
  • 데이터 콘텐츠를 수집하지 않습니다
  • 기본적으로 비활성화되어 있습니다

사용자는 해당 네트워크에 대한 적절한 권한이 있는 경우에만 기능을 사용해야 합니다.

3. 데이터 저장

  • 모든 데이터는 사용자 기기 내부에만 저장됩니다
  • 외부 서버로 자동 전송되지 않습니다
  • 앱 삭제 시 데이터는 함께 삭제됩니다
  • 합리적인 보호 조치를 통해 데이터가 보호됩니다

4. 인앱 결제 (Google Play Billing)

  • 결제는 Google Play 결제 시스템을 통해 처리됩니다
  • 결제 정보는 Google에 의해 관리됩니다
  • 본 앱은 결제 정보를 직접 수집하거나 저장하지 않습니다
  • Google의 개인정보 처리방침이 적용됩니다

5. 수집하지 않는 정보

본 앱은 다음 정보를 수집하지 않습니다:

  • 이름, 이메일, 전화번호
  • 계정 정보
  • GPS 위치 데이터
  • 사진, 영상, 오디오
  • 네트워크 통신 내용
  • 광고 식별자
  • 외부 추적 ID

6. 제3자 공유

  • 사용자 데이터는 제3자에게 판매되거나 공유되지 않습니다
  • 광고 및 분석 SDK를 사용하지 않습니다
  • 법적 요구가 있는 경우에만 제한적으로 제공될 수 있습니다
  • 인앱 결제 시 Google 시스템과의 통신이 발생할 수 있습니다

7. 사용자 권리

  • 데이터 삭제
  • 앱 삭제
  • 권한 거부

8. 아동 개인정보

본 앱은 13세 미만 아동을 대상으로 하지 않습니다.

9. 정책 변경

정책 변경 시 앱 또는 배포 페이지를 통해 안내됩니다.

Privacy Policy

English

This Privacy Policy explains how SafeNet Check (“the App”) processes information.

The App is designed to help users understand their local network environment and does not provide surveillance capabilities.

1. Core Principles

  • No user account is required.
  • No advertising or tracking SDKs.
  • All analysis is performed on-device.
  • No intentional collection of personally identifiable information.
  • No device tracking or persistent identification of specific devices.

2. Information Processed

To provide functionality, the App processes limited technical information as described below.

2.1 Local Network Environment

The App processes limited technical data to understand characteristics of the local network environment, including:

  • Local IP address
  • Gateway information
  • Network type
  • ARP cache data
  • Presence of mDNS / SSDP signals
  • Wi-Fi access point information
Important
  • The content of network communications is never collected or stored.
  • External internet traffic is not monitored or analyzed.
  • The App is not designed to identify or track specific devices.

2.2 BLE Signals

  • RSSI (signal strength)
  • Session-based randomized identifier

Safeguards

  • No real MAC addresses are stored.
  • Device names or service UUIDs are not collected by default.
  • All data remains on-device.
  • Not used to identify individuals or specific devices.

2.3 Optional Feature: Active Discovery

  • Local-only scope
  • User-enabled only
  • No external scanning
  • No content collection
  • Disabled by default

Users should only use this feature in networks where they have proper authorization.

3. Data Storage

  • Stored locally only
  • No cloud upload
  • Removed on uninstall
  • Reasonable safeguards applied

4. In-App Purchases

In-app purchases are handled by Google Play Billing. The App does not collect or store payment data.

5. Information Not Collected

  • Personal identity data (name, email, phone)
  • Account credentials
  • Location history
  • Media files
  • Communication content
  • Advertising identifiers
  • External tracking identifiers

6. Third-Party Sharing

  • No data selling or sharing
  • No analytics or ads SDKs
  • Disclosure only if required by law
  • Google Play Billing communication may occur when purchases are made

7. User Rights

  • Delete stored data
  • Uninstall the app to remove data
  • Deny optional permissions/features

8. Children

Not intended for users under 13.

9. Updates

Significant changes will be communicated in-app or via the distribution platform.
